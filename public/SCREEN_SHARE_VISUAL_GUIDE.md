## Animation Flow

### Screen Share Start
```
1. User clicks "Start Screen Share"
   â†“
2. Browser shows screen selection dialog
   â†“
3. User selects screen/window/tab
   â†“
4. Stream created
   â†“
5. VideoGrid.setScreenShare() called
   â†“
6. Screen share element created with fade-in animation
   â†“
7. Layout automatically switches to speaker view
   â†“
8. Screen share displayed prominently (70% height)
   â†“
9. Other videos moved to thumbnail container (15% height)
```

### Screen Share Stop
```
1. User clicks "Stop Sharing" button
   â†“
2. handleStopSharing() emits event
   â†“
3. App calls screenShareManager.stopSharing()
   â†“
4. VideoGrid.clearScreenShare() called
   â†“
5. Fade-out animation applied
   â†“
6. Video tracks stopped
   â†“
7. Element removed from DOM
   â†“
8. Layout restored to previous mode
   â†“
9. Videos rearranged smoothly
```

- Background: `linear-gradient(180deg, rgba(0,0,0,0.8) 0%, rgba(0,0,0,0) 100%)`

- Border: `rgba(255, 255, 255, 0.2)`

- Stop Button: Red gradient (`#f093fb` to `#f5576c`)

### Screen Share Border
- Main Screen Share: 3px solid Primary Purple (`#6f00ff`)
- Regular Video: No special border

## Summary

This visual guide demonstrates the complete UI/UX flow of the screen share integration. The implementation provides:

- **Clear Visual Hierarchy**: Screen share is prominently displayed
- **Intuitive Controls**: Easy-to-find fullscreen and stop buttons
- **Smooth Animations**: Professional fade in/out effects
- **Responsive Design**: Works on all screen sizes
- **Accessibility**: Keyboard navigation and screen reader support
- **Error Handling**: Clear feedback for all states

The design follows modern UI patterns with glassmorphism effects, smooth transitions, and clear visual indicators for all states.
